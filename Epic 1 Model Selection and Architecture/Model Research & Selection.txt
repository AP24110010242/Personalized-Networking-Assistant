The model selection process is arguably the most consequential step in any AI application. Choosing the wrong model can result in slow responses, poor accuracy, high resource consumption, or output that fails to meet user expectations. For this project, two distinct NLP tasks required model evaluation: event theme classification and conversation text generation.

Theme Extraction: Why DistilBERT Was Chosen

For extracting themes from event descriptions, the team evaluated several Hugging Face transformer models. The primary requirement was the ability to classify text into categories without task-specific training data — a technique known as zero-shot classification. This is critical because the application must work for any type of networking event, not just pre-trained categories.

DistilBERT was ultimately selected because it achieves the optimal balance between inference speed and classification accuracy. For a real-time web application where users expect immediate feedback, a model that responds in milliseconds rather than seconds makes a tangible difference in user experience. Its compact size also makes it far more deployable in resource-constrained environments.
