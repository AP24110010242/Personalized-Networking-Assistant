With all service modules built and tested in isolation, they are wired together through FastAPI's APIRouter in the conversation.py file. This routing layer acts as the integration point between the HTTP interface and the business logic layers, handling request deserialization, service orchestration, and response serialization.

The router defines three POST endpoints, each serving a distinct function. The /analyze-event endpoint provides a standalone theme extraction capability, useful for debugging or building custom integrations. The /fact-check endpoint wraps the Wikipedia service in a type-safe API contract. The /generate-conversation endpoint is the primary application endpoint that orchestrates the full pipeline: it calls the event analyzer to extract themes, passes those themes along with user interests to the topic generator, automatically saves the complete interaction to history, and returns both the topics and suggestions in a single structured response.

The key architectural insight here is that /generate-conversation performs automatic side-effect logging — every successful conversation generation is automatically persisted to history without requiring any additional action from the frontend. This separation of concerns means the frontend does not need to know about history logging; it simply makes a single API call and receives its response.
