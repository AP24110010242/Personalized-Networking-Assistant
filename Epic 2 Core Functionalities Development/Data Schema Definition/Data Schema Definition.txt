Before building any service logic or routes, it is essential to define the data contracts between the frontend and backend. These contracts are expressed as Pydantic models, which provide automatic type validation, serialization, and clear documentation through FastAPI's built-in Swagger UI.

This file defines six Pydantic model classes that form the backbone of all data exchange in the application. By inheriting from BaseModel, each class gains automatic validation — if a required field is missing or has the wrong type, FastAPI will automatically return a 422 Unprocessable Entity error with a clear description of what went wrong, without requiring any additional error-handling code
